Hadoop搭建

发表于 2018-07-31 分类于 机器

Hadoop集群环境搭建

配置描述：

配置环境：

Win7 64bit旗舰版系统

内存：4G（建议至少4内存，虚拟机对内存需求旺盛）

处理器：intel i5-3210m

软件需求：VMWARE-WORKSTATION-12.0（虚拟机环境配置）

SSH client（win7-linux系统文件传输，基于ssh协议）

hadoop-2.6.4.tar.gz（最高版本为2.7.2，为了避免最新版本意外bug与不稳定，我们选择2.6.4版本

CentOS-7-x86_64-DVD-1511.iso 虚拟机镜像版本。（注意是64bit，根据自己计算机实际情况做相应调整）

集群环境描述：

Master Slave0 Slave1

一个虚拟机作为Master（namenode），另外两台是Slave0 ，Slave1（datanode）。并分别以此为虚拟机主机名。

虚拟机配置：内存按照建议的最小内存512M给(后来验证384M也可以跑起来)，磁盘空间默认即可。

虚拟机的安装

虚拟机名字自定，你知道这个是master机器即可，位置默认，下一步。

Cpu设置，默认即可，下一步

内存设置为512M，推荐的最小内存，然后下一步。由于三台虚拟机跑起来还是蛮吃内存的，所以选了最小的。

网络连接桥接，切记。然后下一步

Io控制器选择默认的即可，下一步。

磁盘也是默认，下一步。

磁盘名默认。这是根据你虚拟机名字hadoop_master自动生成的。下一步

配置完毕，完成即可，打开虚拟机。

hadoop_slave0,hadoop_slave1.请自行修改。后面两个虚拟的安装过程不再截图，与此步骤完全相同。

虚拟开始安装并配置：

按上下左右方向，选到默认的Install CentOS 7回车即可

狂按回车：

开始装虚拟机，语言默认即可。别选中文，中文在路径命名的时候会很麻烦。单击continue：

安装选项很多。莫慌，我们只需要配置磁盘选项一个（有红字标识的）。然后就选择安装即可

点进去，把没选的磁盘选了就行，变成蓝色了，然后选择done

最后选择蓝色的begin installation即可。

开始安装读条。在漫长的等待中，我们还需要给root用户创建密码，并创建hadoop用户，hadoop用户的密码自己设置，密码太简单系统会提示你点击两次done来确认。之后就可以等虚拟机安装完成了。

设置root密码：

创建hadoop用户

到这里，虚拟机安装就结束了。

装完了这个样子，系统提示点击reboot重启。

小结：三台虚拟机配置一样，除了名字。创建了相同的hadoop用户，设置了root密码。为了方便我设置了root和hadoop用户相同的密码，总共六个密码都一样。实际环境不推荐这样，系统不安全。

虚拟机基本配置（hadoop加入sudoers表，网络配置，安装net-tools工具，修改hosts文件。三台虚拟机都要配置）：

为了方便的使用虚拟机，我们进行几个简单的系统配置。这就是虚拟机安装完成之后的样子，是命令行版本的，并没有图形操作界面，需要熟悉linux命令行操作。首先配置vmware的网络设置，在菜单栏中-编辑-虚拟网络编辑器

在详细的配置选项中，要选择中间的桥接，并且桥接的适配器要选择所用计算机联网用的适配器。比如笔者用的是有线网的Qualcomm Atheros网卡，如果无线网可能是Intel-wireless网卡，请读者根据实际情况配置。然后应用即可。

用root用户登录进去：

首先修改 /etc/sudoers文件，将我们的hadoop用户加入sudoer列表。这个表示可以临时赋予hadoop用户superuser权限。

Linux命令：vi /etc/sudoers

在sudoers中加入我们的hadoop用户，和root用户相同的配置下面一行是我们自己添加的，原来并没有。

退出保存

Linux命令：Ecs然后wq。

切换到hadoop用户。

Linux命令：su hadoop

调试网络。若果ping www.baidu.com正常则可以跳过这一步，直接去下一步安装net-tools工具

进入目录/etc/sysconfig/network-scripts

Linux命令：cd /etc/sysconfig/network-scripts

列出所有文件：

Linux命令：ls –a

文件只截图了一部分，最重要的我们要修改的ifcfg-eno-xxxxx就是我们的网络配置文件，文件名可以随着不同机器而不同。

修改此文件：vi ifcfg-eno16777736，把最后一行的ONBOOT=no，no改为yes。表示开机启用网络。

然后重启，重启之后我们的网络就可以正常使用了。Linux命令：sudo reboot

使用 ping www.baidu.com 是有正常的icmp报文回传。

网络配置结束。后面安装net-tools工具，便于我们查看网络。

Linux命令：sudo yum install net-tools

按照命令提示，在安装过程中输入y或者yes即可。

安装完了，用ifconfig命令，可以查看虚拟机系统的ip地址和详细信息。记住各个机器ip

最后的配置：根据master和slave0，slave1的ip地址（ifconfig命令可以查看）修改hosts文件和hostname文件。

Linux命令：sudo vi /etc/hosts

修改hostname文件：

Linux命令：sudo vi /etc/hostname

将原来的localhost.localdomin（文件中只有这几个字0.0）改为虚拟机的对应的位置，master或者slave0或者slave1.

在此只演示修改了master的hostname文件，其他两个请自行修改。

修改完成，hosts文件是对应主机名和ip地址的映射，我们做集群实验，可能ip地址会换掉，但是如果在配置是使用主机名代替，那么当ip地址变更时只需要修改hosts文件即可。结果小测试

hostname显示当前主机名为master，而在master上ping，slave0和slave1，表示hosts配置成功。下面就可以开始ssh无密码登录配置了。

S**sh无密码登陆配置**

原理简述：基于ssh协议，利用rsa非对称加密算法加密，将相应的公钥文件放入合适的文件夹即可。若要master无密码登录slave0，需要将密码文件在master和slave0中存放合理即可。最后我们需要做到master无密码登录slave0和slave1，slave0登录master，slave1登录master，共配置四次。在此仅展示一次配置，其他三次请自行操作。

步骤简述：在master生成密码，存到master和slave0相应位置，重启ssh服务即可无密码登录,master就可以无密码登录slave0。

详细步骤：

在master上生成秘钥，注意！使用默认路径，会在/home/hadoop/.ssh目录下生成，而这个秘钥文件在写在/home/hadoop/.ssh/authorized_keys文件中。在生成秘钥过程中会自动生成.ssh目录。我们在把文件拷贝过去即可。

开始：

先在三台机器上开启无密码登陆需要的设置

Linux命令：sudo vi /etc/ssh/sshd_config

然后修改#RSAAuthentication yes和#PubKeyAuthentication yes。把原来的文件最前面的#删除就行了。然后：wq！退出即可。

生成秘钥（hadoop用户登录之后进行操作）

Master机Linux命令：ssh-keygen –t rsa –P ‘’

(注：ssh-keygen中间没有空格，后面之间都有空格，如gen 和 –t之间。-t表示加密算法，采用rsa)

黄色框是提示输入秘钥名字，敲回车就行了，用默认的。然后在~/.ssh目录下就会有秘钥文件，在此使用ls –a命令查看结果如下：

然后把秘钥文件id_rsa.pub追加到.ssh文件夹中的authorized_keys文件中，使用cat命令会直接新建并写入该文件。

Linux命令：cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

更改authorized_keys文件的权限为600

Linux命令：chmod 600 ~/.ssh/authorized_keys,如图结果就是600权限

同时在slave0虚拟机生成秘钥文件，这一步操作是为了后面的做基础，并且这一过程会默认生成相应权限的.ssh目录，避免麻烦。

Linux命令：ssh-keygen –t rsa –P ‘’结果如下：

然后把master机的id_rsa.pub复制到slave0的~/.ssh/authorized_keys文件中

Linux命令：scp /.ssh/id_rsa.pub hadoop@slave0：/.ssh/id_master_keys

在slave0机 中把/.ssh/id_master_keys写入/.ssh/authorized_keys文件中即可。

Linux命令：cat ~/.ssh/id_master_keys >> ~/.ssh/authorized_keys

完成之后，修改authorized_keys文件权限为600

Linux命令：chmod 600 authorized_keys同master，不在截图

最后重启master，slave0的sshd服务：

Linux命令：sudo service sshd restart

最后就可以无密码登录了，master登录slave0.小结：我们把master生成的密码文件在master和slave0中存储，就可以实现master无密码访问slave0。其他三组无密码登录操作与此同不再赘述。

结果如下：

没有输入密码就直接登录了。

JDK配置

(三个虚拟机一样的配置方法，轮流配置)

思路概述：使用windows和linux之间的文件传输工具ssh-client，把windows下载好的jdk文件复制到linux系统中。然后进行相关配置。（建议同时把hadoop文件复制过去）。

默认会把文件存储在/home/hadoop目录下，也就是~。我们去把jdk文件解压一下，然后配置/etc/profile文件即可。

解压压缩包：

Linux命令：tar –xzvf jdk-8u101-linux-x64.tar.gz –C ~/software/

（参数-C表示解压到制定目录，此目录必须提前创建）

Jdk路径配置：

修改/etc/profile文件

Linux命令:sudo vi /etc/profile

添加下面三行：

# set java environment

export JAVA_HOME=/home/hadoop/software/jdk1.8.0_101/

export CLASSPATH=.:$CLASSPATH:$JAVA_HOME/lib:$JAVA_HOME/jre/lib

export PATH=$PATH:$JAVA_HOME/bin:$JAVA_HOME/jre/bin

配置完成之后使用source /etc/profile 使配置文件生效。

java –version查看。配置成功即为下图

（scp –r /software/jdk1.8.0_101 hadoop@slave0:/software）

H**adoop环境配置**

主要分三步：

压缩包解压和目录建立，xml与sh文件的配置，防火墙关闭及hadoop启动。

压缩包解压和目录建立

解压hadoop到/home/hadoop/software目录

Linux命令：tar –xzvf ~/hadoop-2.6.4.tar.gz –C ~/software/

然后修改环境配置/etc/profile文件

​ sudo vi /etc/profile

新添2行

export HADOOP_HOME=/home/hadoop/software/hadoop-2.6.4/

export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin

同样配置完成之后执行source /etc/profile更新配置文件。配置成功hadoop version命令则会显示hadoop版本信息：

然后开始配置hadoop的配置文件。主要分两类.sh的脚本文件和.xml的配置文件。

先在hadoop目录下创建三个目录，tmp,namenode,datanode目录，用来存储相应的数据。

Linux命令：

mkdir ~/software/hadoop-2.6.4/tmp ~/software/hadoop-2.6.4/namenode ~/software/hadoop-2.6.4/datanode

xml与sh文件的配置：

需要配置的文件主要有：hadoop-env.sh,core-site.xml,hdfs-site.xml,mapred-site.xml,slaves.五个文件。

首先配置

hadoop-env.sh

Linux命令：cd ~/software/hadoop-2.6.4/etc/hadoop/

vi hadoop-env.sh

添加JDK的路径：

export JAVA_HOME=/home/hadoop/software/jdk1.8.0_101/

然后是core-site.xml

Linux 命令vi core-site.xml

加上两组属性配置：

​ fs.defaultFS

​ hdfs://master:9000

​ hadoop.tmp.dir

​ /home/hadoop/software/hadoop-2.6.4/tmp

hdfs-site.xml

vi hdfs-site.xml

加上三组属性配置：

​ dfs.namenode.name.dir

​ /home/hadoop/software/hadoop-2.6.4/namenode

​ dfs.datanode.data.dir

​ /home/hadoop/software/hadoop-2.6.4/datanode

​ dfs.replication

​ 2

(后续环境的配置。绿色配置可选)

第四个

mapred-site.xml

（在次配置文件目录没有这个文件，但是有一个模板文件mapred-site.xml.template）

使用 cat mapred-site.xml.template >> mapred-site.xml就可以把模板中的内容复制到mapred-site.xml，然后使用vi命令编辑即可。

加入一行配置即可：

​ mapreduce.framework.name

​ yarn

第五个slaves文件，把原来里面的localhost删除，把slaves节点的主机名slave0和slave1加上即可。

防火墙关闭及hadoop启动

tips：centos7采用了systemctl工具来控制防火墙，而不是曾经的iptables这个services。

使用命令：

sudo systemctl stop firewalld

开始格式化namenode，使用命令：hadoop namenode -format

启动hadoop，linux命令：start-all.sh
